# Requirements

## Requirement IDs

Use `HES-REQ-<DOMAIN>-0001`. IDs are stable, unique, and never reused.

## Traceability

Each requirement should trace to source document sections, ADRs or RFCs, tests, implementation evidence, and release versions.

## Requirement Lifecycle

Requirements move through `Draft`, `Reviewed`, `Accepted`, `Verified`, `Deprecated`, and `Obsolete`.

## Verification

Verification proves the implementation satisfies the requirement through inspection, tests, replay evidence, benchmark evidence, or certification artifacts.

## Validation

Validation confirms the requirement reflects the intended business, operational, safety, and engineering outcome.
