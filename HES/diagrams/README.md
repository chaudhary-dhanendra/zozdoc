# Diagram Repository

This directory stores shared HermesNet diagrams by view type.

## Folders

- `component/`: component boundaries and dependencies.
- `sequence/`: protocol, event, and workflow interactions.
- `state/`: lifecycle and state-machine diagrams.
- `deployment/`: runtime topology and operational placement.
- `memory/`: memory layout, ownership, and lifecycle views.
- `network/`: network paths, segmentation, and connectivity.

## Conventions

Use Mermaid where possible. File names should include domain, view, and version. Diagrams must not be the sole source of normative behavior; the governing document must state requirements in text.
