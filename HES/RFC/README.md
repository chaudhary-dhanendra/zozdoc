# Requests for Comments

## Purpose

RFCs provide a structured path for proposing material changes before they become specification text, ADRs, requirements, or implementation work.

## RFC Lifecycle

- `Draft`: author is preparing the proposal.
- `Discussion`: reviewers are evaluating scope, alternatives, and impact.
- `Accepted`: governance approved the proposal for incorporation.
- `Rejected`: proposal will not proceed.
- `Implemented`: accepted proposal has been incorporated into controlled documents.
- `Archived`: retained for historical traceability.

## Future RFC Content

RFCs should include problem statement, goals, non-goals, proposed design, compatibility, security, performance, operational impact, affected volumes, rollout plan, and open questions.
