# HermesNet Engineering Specification Roadmap

## Current Completion

Estimated repository completion: 50%. Volumes I through V exist as substantive specification content; Volumes VI through X remain planned or placeholder volumes.

## Volumes Completed

- Volume I: Foundation & Core Trading Infrastructure.
- Volume II: Trading Kernel Internals.
- Volume III: Algorithms & Deterministic Execution.
- Volume IV: Wallet, Ledger & Financial Accounting.
- Volume V: Futures, Margin & Liquidation.

## Volumes Remaining

- Volume VI through Volume X require planned content development and review.

## Future Milestones

1. Governance baseline and repository structure.
2. Requirement traceability model.
3. ADR and RFC intake process.
4. Reference tests and golden vectors.
5. Release candidate certification.

## Target Version 1.0

Version 1.0 should provide a stable implementation baseline for core exchange behavior, deterministic replay, accounting correctness, and operational recovery.

## Target Version 2.0

Version 2.0 should incorporate mature certification evidence, expanded operations guidance, advanced risk controls, and implementation conformance suites.

## Implementation Handbook

A future handbook should translate normative HES requirements into implementation sequencing, repository layout, test strategy, and operational rollout guidance.

## Reference Tests

Reference tests should cover deterministic replay, matching, ledger accounting, wallet behavior, margin, liquidation, snapshots, journals, and recovery.

## Engineering Portal

A future engineering portal should publish stable documentation, generated diagrams, ADRs, RFCs, requirements, and release evidence.
